# 产品路线图

本路线图用于描述高考志愿推荐网站从需求阶段到正式上线的建设节奏。

## 阶段一：需求与方案

目标：明确产品定位、核心用户、推荐逻辑、数据需求和部署要求。

主要任务：

- 完成一般需求文档
- 明确用户流程
- 明确 MVP 范围
- 明确 AI 接入边界
- 明确国内部署和合规要求
- 确定首发省份策略

交付物：

- 需求文档
- README
- 部署说明
- 路线图

## 阶段二：PRD 与原型

目标：将一般需求细化为研发可执行的产品需求。

主要任务：

- 首页和输入表单设计
- 推荐结果页设计
- 院校详情页设计
- 专业详情页设计
- 对比功能设计
- 报告导出流程设计
- 后台数据导入流程设计
- 异常状态和空状态设计

交付物：

- PRD
- 页面流程图
- 字段说明
- 交互原型

## 阶段三：数据模型与规则

目标：建立可落地的推荐数据结构和规则口径。

主要任务：

- 院校表设计
- 专业表设计
- 招生计划表设计
- 历年录取数据表设计
- 一分一段表设计
- 招生章程表设计
- 推荐评分规则设计
- 冲稳保分档规则设计

交付物：

- 数据字典
- 推荐规则文档
- 数据导入模板
- 数据质量检查规则

## 阶段四：MVP 开发

目标：完成可演示、可内测的最小可用版本。

主要任务：

- 前端首页和推荐表单
- 推荐结果页
- 筛选和排序
- 院校专业对比
- PDF 报告导出
- 后台数据导入
- 基础权限管理
- MVP 部署

验收标准：

- 用户可以输入基本信息并获得推荐结果。
- 推荐结果可以展示冲、稳、保、兜底分层。
- 每条推荐可以展示推荐理由和风险说明。
- 管理员可以导入核心数据。
- 网站可在中国大陆无需代理访问。

## 阶段五：正式上线

目标：完成公开访问、备案、监控和高峰期准备。

主要任务：

- 完成 ICP 备案
- 完成公安联网备案
- 配置 HTTPS
- 配置 CDN
- 配置云数据库和备份
- 配置监控告警
- 优化高峰期性能
- 完善隐私政策和用户协议

验收标准：

- 网站稳定运行。
- 推荐结果响应时间满足目标。
- 数据来源和更新时间可追溯。
- 基础合规信息完整展示。
- 高峰期有扩容预案。

## 阶段六：智能化增强

目标：在不影响推荐可信度的前提下接入 AI 能力。

主要任务：

- 建设专业知识库
- 建设招生章程知识库
- 接入国内可访问 AI 模型服务
- 增加专业解读
- 增加院校章程摘要
- 增加志愿报告润色
- 增加问答助手
- 评估 AI 和算法合规要求

验收标准：

- AI 输出有依据来源。
- AI 不直接承诺录取结果。
- 核心推荐仍由规则系统生成。
- 用户可以区分数据结论和 AI 解读。

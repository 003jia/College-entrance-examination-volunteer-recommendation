# 国内部署与合规说明

本项目要求网站在中国大陆无需代理即可访问。部署方案应优先选择中国大陆云服务和国内可访问的基础设施。

## 部署目标

- 中国大陆访问稳定
- 高考季高峰期可扩容
- 数据安全可控
- 满足网站备案和基础合规要求
- 不依赖需要代理访问的海外服务

## 推荐云服务

可选择以下云厂商之一：

- 阿里云
- 腾讯云
- 华为云

推荐地域：

- 上海
- 杭州
- 北京
- 广州
- 成都

## MVP 部署方案

适用于早期验证、小范围内测和低并发访问。

建议配置：

- 服务器：2 核 4G 轻量应用服务器
- 系统：Ubuntu LTS 或 Alibaba Cloud Linux
- Web：Nginx
- 后端：Node.js/NestJS、Java Spring Boot 或其他团队熟悉的框架
- 数据库：PostgreSQL 或 MySQL
- 文件存储：本地存储或对象存储
- HTTPS：免费 DV SSL 证书

优点：

- 成本低
- 部署简单
- 适合快速验证产品方向

限制：

- 高峰期承载能力有限
- 数据库和应用同机存在单点风险
- 运维和备份需要额外配置

## 正式部署方案

适用于公开上线和商业化运营。

建议架构：

- 前端：对象存储 + 国内 CDN
- 后端：至少 2 台云服务器或容器服务
- 网关：负载均衡
- 数据库：云数据库 MySQL/PostgreSQL 主备版
- 缓存：Redis
- 文件：对象存储
- 安全：WAF、防火墙、DDoS 基础防护
- 运维：日志服务、监控告警、自动备份

## 不建议依赖的服务

核心链路不建议依赖：

- Vercel
- Netlify
- GitHub Pages
- Cloudflare Pages
- Google Fonts
- Google Analytics
- 境外 CDN
- 需要代理访问的海外 AI API

如确需使用境外服务，应确保不影响中国大陆用户访问主流程。

## 域名与备案

上线中国大陆服务器通常需要：

1. 注册并实名认证域名。
2. 购买中国大陆地域云服务器。
3. 提交 ICP 备案。
4. 备案通过后正式绑定域名开放访问。
5. 网站开通后按要求办理公安联网备案。
6. 在网站底部展示 ICP 备案号和公安备案号。

如果平台提供收费信息服务，还需要进一步评估是否涉及 ICP 经营许可证等资质。

## 安全要求

- 全站启用 HTTPS。
- 管理后台必须登录。
- 后台账号支持强密码和权限分级。
- 敏感操作记录审计日志。
- 数据库定期备份。
- 文件上传限制格式和大小。
- 防范 SQL 注入、XSS、CSRF 和越权访问。
- 用户数据按最小必要原则采集。

## AI 服务要求

如果后续接入 AI：

- 优先选择国内可稳定访问的模型服务。
- AI 仅作为解释、摘要和问答助手。
- 核心推荐结果仍由结构化数据和规则引擎生成。
- AI 输出应基于知识库和平台数据。
- 需要评估算法推荐、生成式 AI、内容安全和投诉机制要求。

## 上线检查清单

- ICP 备案完成
- 公安联网备案准备完成
- HTTPS 证书配置完成
- 网站底部备案信息展示完成
- 隐私政策和用户协议上线
- 数据来源和更新时间可展示
- 推荐结果免责声明上线
- 数据库备份策略配置完成
- 监控告警配置完成
- 高峰期扩容方案准备完成

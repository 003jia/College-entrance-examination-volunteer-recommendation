# 高考志愿推荐系统（MVP）

一个面向高考考生、家长和升学指导场景的志愿辅助决策网站。用户输入省份、分数、位次、选科组合、目标专业和个人偏好后，系统基于位次法给出可解释的高校与专业建议，并按照“冲、稳、保、兜底”进行风险分层。

> 当前数据库内为演示样例数据，位次与分数不是真实录取数据。本项目定位为志愿填报辅助工具，不替代各省教育考试院、教育部阳光高考平台和高校官方招生系统，不承诺录取结果。

## 技术栈

React 19 + Vite + TypeScript + Express 5 + node:sqlite + Zod + Vitest

## 快速开始

```bash

npm run dev
```

- API：`127.0.0.1:8787`
- 前端：由 Vite 输出本地访问地址

常用命令：

```bash

npm run typecheck
npm run build
npm run db:seed
```

如果需要避免 API 与本地预览器共用 `PORT` 变量，可使用：

```bash

```

## 核心目标

- 根据高考分数、位次、选科和目标专业给出高校建议。
- 用位次和历年录取数据做核心判断，避免只按分数粗略匹配。
- 输出推荐依据，让用户理解风险来源。
- 支持中国大陆无需代理访问。
- 为后续 AI 问答、报告生成和志愿咨询服务预留空间。

## 接入真实数据

数据来源建议优先使用官方公开渠道：

| 数据 | 来源 |
| --- | --- |
| 一分一段表 | 省教育考试院官网 |
| 历年录取位次/分数 | 省考试院平行投档分数线表 |
| 招生计划 | 教育部阳光高考平台 + 省考试院年度计划 |
| 章程限制 | 阳光高考招生章程库 + 高校招生官网 |

导入流程：

```bash
npm run import:segments -- --file 浙江2026一分一段.csv --dry-run
npm run import:segments -- --file 浙江2026一分一段.csv

npm run import:admissions -- --file 浙江投档线.csv --dry-run
npm run import:admissions -- --file 浙江投档线.csv --version "浙江 2023-2025 投档线"
```

导入模板：

- [data/templates/score-segments-template.csv](data/templates/score-segments-template.csv)
- [data/templates/admissions-template.csv](data/templates/admissions-template.csv)

设置环境变量 `GAOKAO_ADMIN_TOKEN` 后，后台导入接口会要求请求头 `x-admin-token` 匹配；未设置时本地开发默认放行。

## 推荐逻辑原则

推荐系统应以“位次”为核心，而不是单纯使用分数。

基础流程：

1. 按省份、年份、科类或选科组合过滤数据。
2. 按目标专业、专业类、选科要求和招生限制过滤不符合条件的记录。
3. 对比用户位次与近年最低录取位次。
4. 结合招生计划变化、专业热度、城市偏好和院校层次进行评分。
5. 输出冲、稳、保、兜底建议。
6. 为每条建议展示风险等级和推荐依据。

## AI 接入原则

MVP 阶段不建议让 AI 直接决定推荐结果。核心推荐应由结构化数据和规则引擎完成，确保可解释、可复核。

AI 更适合用于：

- 专业介绍和就业方向解读
- 招生章程摘要
- 推荐理由自然语言生成
- 志愿报告润色
- 基于知识库的问答助手

AI 不应直接输出无依据的录取概率，也不应做“保录取”承诺。

## 国内访问与部署

项目应部署在中国大陆云服务环境，避免核心链路依赖需要代理访问的海外服务。

建议方案：

- 云厂商：阿里云、腾讯云或华为云
- 前端：对象存储 + 国内 CDN
- 后端：云服务器或容器服务
- 数据库：云数据库 MySQL/PostgreSQL
- 缓存：Redis
- 安全：HTTPS、WAF、日志审计和备份

详细说明见 [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md)。

## 后续建设方向

1. 细化 PRD：页面结构、字段规则、交互状态和异常处理。
2. 设计数据模型：院校、专业、招生计划、录取位次、一分一段表。
3. 确定首发省份：建议先选 1-3 个省份试点。
4. 搭建 MVP：完成推荐表单、结果页、对比和报告导出。
5. 完成备案和部署：保证中国大陆访问稳定。
6. 评估 AI 接入：先做解释和问答，不做核心决策。

## 合规提示

- 上线中国大陆服务器前需要完成 ICP 备案。
- 网站开通后应按要求完成公安联网备案。
- 如提供收费信息服务，需进一步评估 ICP 经营许可证等资质要求。
- 如引入算法推荐或生成式 AI 服务，需进一步评估相关备案和内容安全要求。
- 平台应明确提示：所有建议仅供参考，最终以官方招生政策和志愿填报系统为准。

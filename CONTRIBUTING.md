# 贡献说明

本仓库当前处于需求设计和产品方案阶段。贡献内容应优先围绕需求准确性、数据口径、合规风险和产品可落地性展开。

## 适合贡献的内容

- 产品需求补充
- 用户流程优化
- 推荐规则建议
- 数据结构建议
- 国内部署方案优化
- 合规风险补充
- 文档错别字和表述修正

## 文档约定

- 主要说明文档使用中文。
- 产品概念、风险等级和推荐口径应保持一致。
- 涉及招生政策、录取数据、备案要求时，应尽量标注来源和更新时间。
- 不使用“包录取”“保录取”“必中”等确定性表述。
- 不把 AI 输出作为最终录取判断依据。

## 提交流程

1. 基于最新 `main` 分支修改。
2. 保持改动范围清晰，避免一次提交混入无关内容。
3. 提交前检查 Markdown 链接、标题层级和表格格式。
4. 提交信息应简短说明改动内容。

示例：

```bash
git add README.md docs/ROADMAP.md
git commit -m "Update project documentation"
```

## 评审重点

- 是否符合高考志愿辅助工具定位
- 是否避免录取承诺和误导性表达
- 是否能够在中国大陆稳定访问
- 是否保留数据来源和更新时间意识
- 是否便于后续研发落地
